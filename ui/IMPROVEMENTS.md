# 프런트엔드 코드 개선 사항

## 발견된 문제점 및 개선 사항

### 1. 🔴 중요: 중복된 Alert 메시지
- **문제**: `Cart.jsx`와 `App.jsx`에서 모두 alert를 호출하여 중복 표시됨
- **위치**: `Cart.jsx:22`, `App.jsx:120`
- **해결**: Cart 컴포넌트의 alert 제거 (App.jsx에서만 처리)

### 2. 🟡 사용하지 않는 Import
- **문제**: `App.jsx`에서 `useEffect`를 import했지만 사용하지 않음
- **해결**: 불필요한 import 제거

### 3. 🟡 키 값 문제
- **문제**: `Cart.jsx`에서 배열의 index를 key로 사용 (React 권장사항 위반)
- **해결**: 고유한 ID 조합을 key로 사용

### 4. 🟡 에러 처리 부족
- **문제**: 
  - 주문 생성 시 빈 장바구니 체크는 있지만, 다른 에러 케이스 처리 부족
  - 재고 부족 시 주문 제한 로직 없음
- **개선 제안**: 재고 확인 로직 추가 (선택사항)

### 5. 🟢 사용자 경험 개선
- **개선 제안**:
  - 장바구니에 추가 시 시각적 피드백 (예: 토스트 메시지)
  - 주문 완료 시 alert 대신 더 나은 피드백 (선택사항)

### 6. 🟢 접근성 개선
- **개선 제안**:
  - 버튼에 aria-label 추가
  - 키보드 네비게이션 지원

### 7. 🟢 성능 최적화
- **현재 상태**: 대시보드 통계는 매 렌더링마다 계산됨 (작은 데이터셋이므로 문제 없음)
- **개선 제안**: useMemo로 최적화 가능 (선택사항)

## 즉시 수정할 항목

1. ✅ 중복 alert 제거
2. ✅ 사용하지 않는 import 제거
3. ✅ 키 값 개선

## 향후 개선 제안

- 재고 부족 시 주문 제한 기능
- 더 나은 사용자 피드백 (토스트 메시지 등)
- 접근성 개선
- 성능 최적화 (필요시)
